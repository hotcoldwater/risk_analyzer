# AuditRisk-AI Design Guidelines

## 목적

제공된 대시보드 레퍼런스를 분석해 이 프로젝트의 프론트엔드 디자인 기준으로 사용한다.

## 제품 톤

- 회계, 감사, 재무 분석 서비스다운 전문성과 절제를 유지한다.
- 과한 장식보다 정보 위계와 신뢰감을 우선한다.
- 소비자용 앱보다 엔터프라이즈 대시보드에 가깝게 설계한다.

## 레이아웃 원칙

- 기본 구조는 `사이드바 + 상단 바 + 주요 콘텐츠` 조합을 따른다.
- 메인 콘텐츠는 `검색 영역 -> 핵심 결과 카드 -> 보조 정보 카드` 흐름으로 배치한다.
- 데스크톱에서는 넓은 카드와 패널을 사용하고, 모바일에서는 단일 컬럼으로 자연스럽게 접는다.

## 정보 위계

- 가장 중요한 수치는 `부채비율`이다. 가장 큰 크기와 가장 강한 대비를 준다.
- 그 다음 레벨은 기업명, 대상연도, 기업코드, 데이터 출처다.
- 부채와 자본은 보조 지표 카드로 정리한다.
- warning과 에러는 결과 영역 아래 또는 옆에서 명확히 분리해 보여준다.

## 시각 스타일

- 밝은 배경을 기본으로 한다.
- 블루 계열을 핵심 브랜드 색으로 사용한다.
- 회색 계열 배경과 경계선을 섞어 데이터 서비스 느낌을 만든다.
- 상태 표시는 색으로 구분하되, 빨강은 오류나 높은 위험에만 제한적으로 사용한다.

## 컴포넌트 패턴

### 검색 영역

- 페이지 중앙 상단에 배치한다.
- 입력창과 분석 버튼은 한 줄에 두되 모바일에서는 세로로 정렬한다.
- 최근 조회 또는 힌트 텍스트를 보조 정보로 둘 수 있다.

### 결과 카드

- 가장 먼저 기업명, corp code, 대상연도, 출처를 보여준다.
- `부채비율`은 별도 하이라이트 카드로 분리한다.
- 부채와 자본은 동일한 وزن의 카드로 나란히 둔다.
- `cached`, `source`, `warnings`는 배지나 보조 카드로 표시한다.

### 상태 표현

- 로딩은 대시보드 문맥에 맞게 “분석 중” 상태 카드 또는 버튼 상태로 표현한다.
- 에러는 전체 화면을 막지 말고 결과 패널 내에서 친절한 문장으로 안내한다.
- warning은 단순 텍스트보다 아이콘과 배경색을 가진 카드로 보여준다.

## 타이포그래피

- 제목은 강한 산세리프를 사용하되 숫자와 지표는 명확하게 읽혀야 한다.
- 코드, 기업번호, API 관련 보조 정보는 모노스페이스 스타일을 허용한다.
- 한 화면에 지나치게 많은 폰트 변형을 두지 않는다.

## 인터랙션

- 버튼과 카드 hover는 미세한 강조 정도만 사용한다.
- 로딩 시 버튼 비활성화와 진행 메시지를 함께 보여준다.
- 결과가 나오면 자연스럽게 결과 섹션으로 시선이 이동하도록 한다.

## MVP 적용 범위

MVP에서는 다음 요소를 우선 구현한다.

- 브랜드 헤더
- 검색 입력
- 분석 버튼
- 로딩 상태
- 결과 카드
- warning 영역
- 에러 메시지

다음 요소는 레퍼런스 참고만 하고 MVP에서는 더미 또는 축소 구현한다.

- 다중 메뉴 네비게이션
- PDF export
- Share report
- Market context 이미지
- 운영 메트릭 바
